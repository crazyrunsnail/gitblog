# [新工作感想](https://github.com/crazyrunsnail/gitblog/issues/3)

2026年4月，我离开了工作近8年的公司，离开的时候我没有过多的犹豫和不舍。在过去的一年中，我都在寻找机会，面了5次了，收到了3个offer，而这是第3个。前面2个的其中一个是汇丰外包，到了deadline我拒绝了，另外一个是我答应了，他们过了20天左右通知我岗位需要重新评估

在这段求职的过程有几个是比较难接受的: 未满一年没有年假、公积金按基数6K的5%来缴纳 以及 大小周

现在回想起来之前的公司是小而美的公司:
- 试用和转正的待遇一样
- 当月工资当月发
- 第一年就有年假
- 公积金是工资全额的12%
- 每年两次的调薪机会和近2个月的年终
- 不用写CSS

但近几年公司接连操作让我想走: 
- 拿出20%的工资做绩效，基本上是做多扣多
- 4年多没有调薪
- 公积金由原来的12% 降到 5%
- 没有体检

新的公司不再是传统的纯软件和互联网公司，有自己的实体业务，信息技术赋能业务，公司的总规模是比之前的公司大。但负责信息技术加上面试我的负责人才5个人，实际干活的4个人: 副总监什么都懂，主要是系统日常维护，公司其他同事都熟悉; 一个管机房，电脑，网络，虚拟机的同事; 一个大数据开发工程师 还有我。

选择这家公司我没有多过犹豫，在填面试表时我写了JD上工资的最高值，面试过HR爽快地答应了。试用和转正工资一样。第一年也有年假，虽然是现在的一半。

进去发现一些和面试不一样的地方，之前介绍这个团队要8-10人，但我听到风声是后面不招人; 之前的面试官和负责人是海归，进来后发现原来还是老板的二公子，分管了其他了其他部门。最让我觉得比较难接受的了是一定要每在上够8个小时，早9晚6点半。这对我来的精力来讲是一个挑战

新公司大部多很多核心的系统不是自研的，自研的系统业务场景比较简单，而且部分要被其他的第三方系统取代。接手后要维护的主要还是各系统之间的对接工程。虽然代码有点难度，代码质量也不高。借助AI下这些都可以慢慢优化。并发/数据量也不是问题，更多的是要去熟悉业务流程

新公司的开发流程和之前公司开发流程有很大的不同，之前的客户是公司外的客户，有商务、项目经理、产品经理去了解需求和评估，然后再画出原型再写代码再上线。但现在需要我去了解现在的系统，主动去问业务还有一些技术的问题。从之前的技术构建转变成一个全流程的系统构建。但好消息也是数据量不大，技术上压力不大，但需要多方面去沟通去把需求给理解清楚

这些我都能够处理，只要给够时间。但更重要是的个人的发展问题，在这里技术可能得不到成长，如果我脱离了这家公司，后面到了40+，核心竞争力又在哪里？

有时候觉得这可能也是伪命题，如果我还呆在原来的公司，后面年龄上来的我就有竞争力了吗？答案可能是一样的：没有

现在需要关注的，还是要怎么去高素质完成现在的工作，从长期来去规化系统，并将自己做的工作产品化，努力先去度过这个试用期



---

上班的第一天，副总监给了我一台笔记本电脑发和了一张图片各系统的概览。大部多很多核心的系统不是自研的，自研的系统都放在这台电脑里，eclipse 放的是 Java 项目， HBuilderX 放的是终端的项目， 几个简单业务场景和同步工程。

后面我发现不少问题:

- 代码没有版本控制，有svn但没有用，
- 项目虽然都是Java，部分技术都比较旧 war + springmvc + spring，有几个还没用到构建工具，依赖的是 eclipse 传统构建
- 发布比较混乱: 1. 直接 `java -jar xx.jar`（没有jvm参数）2.  tomcat 下的 java web 3. docker 启动 tomcat 但还是挂载路径，路径放的是 java web 的class
- 没有配置管理， 也没有测试环境

虽然有这些问题，但好消息的是业务不复杂，涉及的表结构不多，并且相对稳定不需要很多修改

看完我就知道了我需要做一些事情:
- 搭建一个CI/CD: gitlab, jenkins, harbor
- 搭建单机可拓展的 k3s 集群

在这中间和大模型聊了很多，日常打开 gemini, Claude, ChatGPT, Deepseek, 聊得比较多是从长期去考虑，要怎么去搭建这些基建平台，这过程中间的过程我还是比较享受的
